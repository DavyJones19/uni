--- Conceptos rápidos (antes de los archivos) ---

• Next.js App Router: carpetas bajo `app/` definen rutas. `app/page.tsx` = ruta "/".
• Client Component: archivo con "use client" — React corre en el navegador (hooks, clicks).
• Route Handler: `app/api/.../route.ts` — funciones GET/POST que corren en el SERVIDOR
  (Node). Ahí puedes usar variables de entorno secretas y llamar APIs externas.
• Arrow function (función flecha): `const suma = (a, b) => a + b` — forma corta de
  funciones, útil como callback (p. ej. `.map(x => x * 2)`).
• async/await: `async function f() { const r = await fetch(...) }` — await espera
  la Promise sin bloquear la interfaz en el cliente.
• TypeScript `type` / `interface`: describen formas de datos para el compilador.
• Paginación (en este proyecto): los alumnos llegan todos en un array desde la API;
  la tabla **parte el array en el cliente** (`slice`) y muestra "Mostrando X–Y de Z"
  y "Página N de M". No es paginación en el servidor (no hay `?page=` en la API).

ARCHIVO: app/page.tsx  (página principal — Client Component)
================================================================================
Hay comentarios didácticos también en el propio código fuente.

"use client" — marca el archivo como Client Component (sin esto, no usarías
  useState/useEffect en este archivo).

Imports: React, `LoginForm`, `GrupoSelector`, `TablaAlumnos`, y el **tipo**
  `ColumnDef` desde `DataTablePriority` para tipar los arrays de columnas.

`type RowData = Record<string, unknown>` — cada fila es un objeto con claves string.

`renderNombreCompleto(row)` — función reutilizada en las columnas "Nombre completo"
  de la tabla completa y de la vista simple (evita duplicar lógica).

`RESUMEN_PDF_URL` + `handleDescargarResumen(row)` — abre en nueva pestaña el PDF
  `https://mistalentos.mx/FORMATOS/Talentos_{id_usuario}.pdf` usando `id_usuario`
  de la fila (el campo puede estar oculto como columna pero sigue en `row`).

`DEFAULT_COLUMNS` — plantilla de la **tabla completa**: Grupo, nombre completo,
  Usuario, contraseña, correo, fechas, métricas, tests, columna **DESCARGAR RESUMEN**
  con `render` que muestra un botón solo si `ind_terminado` es 1 / "1" / true.

`DEFAULT_COLUMNS_VISTA_SIMPLE` — plantilla de la **segunda tabla** (demo): solo
  Grupo y Nombre completo (mismo `renderNombreCompleto`). No incluye PWD ni el
  botón de resumen: **no se "ocultan" con `HIDDEN_COLUMN_IDS`**, simplemente **no
  están en esta plantilla**.

`HIDDEN_COLUMN_IDS` — `Set` de **nombres de propiedad** de la API que no quieres
  como columna automática (ni duplicada). Ej.: `AP`, `AM`, `NOMBRE`, `id_usuario`,
  `ind_terminado`, etc. No borra datos del objeto `row`; el `render` del botón
  y `renderNombreCompleto` siguen leyendo esos campos. **Las dos tablas pueden
  compartir el mismo `Set`**: no "chocan" porque `buildColumns` se llama dos veces
  y solo **lee** el Set; cada llamada devuelve un **array de columnas distinto**.

`buildColumns(rows, defaults, hidden, incluirExtras?)`
  • A partir de `defaults` arma `fromConfig` (map con `accessorKey`, filter por
    `hidden`).
  • Si **`incluirExtras` es false** (vista simple): devuelve **solo** `fromConfig`.
    Ahí es por lo que no aparecen PWD ni columnas extra: **no están en `defaults`**
    y no se añaden extras.
  • Si **`incluirExtras` es true** (tabla completa): además añade columnas extra
    por claves de `rows[0]` que no estén en `existingIds` ni en `hidden`.

`normalizeClientGroups`
  Convierte respuestas heterogéneas (string, número, objeto) a `GrupoOption[]`.
  `??` (nullish coalescing): usa el valor de la derecha solo si la izquierda es
  null o undefined.
  `Set` evita duplicados por `value`.

`export default function Home()` — Next usa este componente en la ruta "/".

  useState — token, grupos, `rows` (alumnos del grupo), flags de carga, error, etc.

  Dos `useMemo` sobre el mismo `rows`:
    • `columns` = `buildColumns(rows, DEFAULT_COLUMNS, HIDDEN_COLUMN_IDS, true)`
    • `columnsVistaSimple` = `buildColumns(rows, DEFAULT_COLUMNS_VISTA_SIMPLE, …, false)`
  Así una sola carga de datos alimenta **dos tablas** con columnas distintas.

  `Boolean(token)` — sesión iniciada o no.

  useEffect(..., [token])
         Efecto: cuando `token` cambia, si está vacío limpia grupos; si hay token,
         define `loadGroups` (async), hace fetch a `/api/grupos` con cabecera
         `Authorization: Bearer ...`, parsea JSON y actualiza estado.
         `AbortSignal.timeout(30000)` — cancela la petición si pasa de 30 s.
         `try / catch / finally` — maneja errores y quita loading siempre.

  handleLogin — POST a `/api/login` con `{ Usuario, Pwd }`; lee varias claves
         posibles de token en la respuesta.

  useCallback `loadRows` — POST a `/api/alumnos` con Bearer; rellena `rows`.

  handleGroupChange / useRef + handleSearchClick — grupo seleccionado y búsqueda.

  JSX (en **este** archivo `page.tsx`, no solo en `TablaAlumnos.tsx`)
         Sin token → `<LoginForm />`. Con token → `<GrupoSelector />` y **dos**
         bloques `<section>`: "Tabla completa" con
         `<TablaAlumnos columns={columns} data={rows} />` y "Vista simple" con
         `<TablaAlumnos columns={columnsVistaSimple} data={rows} pagination={false} />`.
         Mismos `rows`, distintas props `columns`. Fragment `<>...</>` agrupa
         sin un `<div>` extra. Opciones de paginación: ver componente.

ARCHIVO: components/DataTablePriority.tsx
================================================================================
Propósito: tabla HTML con columnas ordenadas por `priority` y columnas extra en
           fila expandible en móvil/escritorio; **paginación de filas en el cliente**.

Columnas (definición y celdas):
  ColumnDef<TData> — id, header, accessorKey opcional, render opcional.
  getRowId — identificador estable por fila (id, ID_USUARIO, etc.) o fallback
             `row-${index}`. Con paginación, el `index` debe ser el índice **global**
             en el array `data` (no el índice dentro de la página), para que las
             `key` de React no se repitan entre páginas.
  getCellValue — si hay `render`, lo usa; si no, lee la propiedad del row.

Layout responsive de columnas (independiente de la paginación):
  useState: Set de filas expandidas; isMobile; maxVisible columnas visibles.
  useEffect en mount: window.innerWidth → maxVisible 3 / 5 / 8 según breakpoints;
        cleanup del listener resize.
  orderedColumns — orden por priority; visibleColumns / hiddenColumns — slice.
  toggleExpanded — abre/cierra celdas ocultas en una segunda fila.

Paginación (props opcionales, ver tipo DataTablePriorityProps):
  • `pagination` (default true) — si false, `data` completo sin `slice`.
  • `defaultPageSize` (default 10) — tamaño inicial de página.
  • `pageSizeOptions` — opciones del <select>; si no se pasa, usa [5, 10, 25, 50].

Estado y efectos ligados a la paginación:
  • `pageSize` y `currentPage` en useState.
  • Al cambiar `data` (nueva carga de alumnos), `currentPage` vuelve a 1.
  • `totalPages` = ceil(total / pageSize); `safePage` acota la página actual al
    rango válido; otro useEffect ajusta la página si `totalPages` baja (p. ej. al
    cambiar "filas por página").
  • `start` / `end` — rango [inclusive] para el texto "Mostrando X–Y de Z".
  • `paginatedData = data.slice(start, end)` — solo lo que se pinta en el <tbody>.
  • Barra inferior: texto "Mostrando … de …", "Página N de M", selector de filas
    por página, botones Anterior / Siguiente.

FLUJO RESUMIDO (quién llama a quién)
================================================================================
1. Usuario abre "/" → `app/page.tsx` renderiza LoginForm o tablas según token.
2. Login → fetch `/api/login` → `route.ts` → API externa → token en estado.
3. Con token → useEffect carga `/api/grupos` → selector se llena.
4. Elegir grupo / Buscar → fetch `/api/alumnos` → `setRows` → dos `useMemo` generan
   `columns` y `columnsVistaSimple` → dos `<TablaAlumnos />` con el mismo `rows`.
   La tabla completa usa paginación en cliente ("X–Y de Z", "Página N de M"); la
   vista simple puede ir sin paginación (`pagination={false}`).
